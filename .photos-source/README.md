# Raw Project Photos

Put original phone or camera photos here first.

Why this folder is hidden:
- Firebase is configured to host from the project root.
- Hidden folders are ignored during deploy by the current hosting config.
- That lets us review raw images without publishing every original photo.

Recommended workflow:
1. Drop new originals into `in-progress/` or `completed/`.
2. Review them for blur, lighting, clutter, duplicates, and credibility.
3. Export only the strongest web-ready images into `images/projects/`.
4. Use those curated images in the live site.

Photos that usually do not belong on the site:
- blurry or noisy photos
- very dark images
- harshly overexposed images
- repeated shots from almost the same angle
- messy frames where trash, tools, or personal items overpower the project
- weak finished shots that make the work look worse than it is
